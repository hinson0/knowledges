## 一、Python相关

### 1. Flask中的请求上下文和应用上下文分别是什么？

- **请求上下文**：保存单个请求相关的信息，包含 `request` 和 `session` 对象。请求开始时压入栈，结束时弹出。
- **应用上下文**：保存应用级别的信息，包含 `current_app` 和 `g` 对象。用于在视图函数或离线脚本中访问应用配置和共享数据。

### 2. 谈谈对元类编程的理解

元类是创建类的“类”，即 `type` 或其子类。通过元类可以拦截类的创建过程，动态修改类的属性、方法或实现单例、ORM等模式。例如 `class Meta` 在 Django 中就是利用元类。

### 3. 绿色协程是什么？

绿色协程（Greenlet）是一种轻量级的用户态线程，由程序自身调度而非操作系统内核。协程之间切换开销极小，适合高并发的 I/O 密集型任务。Python 中 `gevent`、`asyncio` 都基于类似概念。

### 4. Python字典的key有什么特征？

- 不能重复（相同 hash 值的键会覆盖）
- 必须是**不可变**且**可哈希**的类型：如字符串、数字、元组（元组内元素也需不可变）
- 列表、字典、集合等可变类型不能作为 key

### 5. 谈谈装饰器，说一下实际工作中的用法

装饰器是一个接受函数并返回新函数的可调用对象，用于在不修改原函数代码的情况下增加额外功能。
**实际用法**：

- 日志记录、权限校验、计时统计
- 缓存（如 `@lru_cache`）
- 路由注册（Flask 中 `@app.route`）
- 事务管理、重试机制

### 6. 生成器和迭代器的关系

- **迭代器**：实现了 `__iter__` 和 `__next__` 方法的对象，可被 `for` 循环遍历。
- **生成器**：是一种特殊的迭代器，使用 `yield` 关键字定义函数，或生成器表达式。生成器自动实现迭代器协议，且惰性求值。
- **关系**：每个生成器都是迭代器，但迭代器不一定是生成器。

### 7. Django和Flask的区别

| 特性     | Django                   | Flask             |
| -------- | ------------------------ | ----------------- |
| 定位     | 大而全的框架             | 轻量级、灵活      |
| 内置组件 | ORM、Admin、表单、认证等 | 极简，需扩展      |
| 学习曲线 | 较陡                     | 平缓              |
| 适合场景 | 大型复杂项目             | 微服务、原型、API |

### 8. Django中F查询和Q查询的区别

- **F查询**：用于对模型字段自身的值进行操作（如 `update(age=F('age')+1)`），避免竞态条件。
- **Q查询**：用于构造复杂的查询条件（如 OR、AND、NOT），支持逻辑组合。

### 9. Django中的DRF继承了哪些类？

常用基类：

- `APIView`：所有视图的基类，类似 Django 的 `View`。
- `GenericAPIView`：提供常用属性（`queryset`、`serializer_class`）和钩子。
- 具体视图：`ListCreateAPIView`、`RetrieveUpdateDestroyAPIView` 等。
- `ViewSet`、`ModelViewSet` 等。

### 10. Python如何实现多进程、多线程、协程？

- **多进程**：`multiprocessing` 模块，利用 CPU 多核，适合 CPU 密集型。
- **多线程**：`threading` 模块，受 GIL 限制，适合 I/O 密集型。
- **协程**：`asyncio` 或 `gevent`，单线程异步并发，适合高 I/O。

### 11. 使用过哪些协程框架？了解过FastAPI吗？

- 协程框架：`asyncio`、`gevent`、`tornado`、`aiohttp`。
- **FastAPI**：现代高性能 Web 框架，基于 `Starlette` 和 `Pydantic`，使用 `async/await` 语法，自动生成 OpenAPI 文档。

## 二、通用技能

### 1. 请说一下操作系统的三级缓存

CPU 三级缓存（L1、L2、L3）用于缓解 CPU 与内存的速度差异：

- **L1**：核内私有，容量最小（几十 KB），速度最快。
- **L2**：核内私有或共享，容量几百 KB。
- **L3**：多核共享，几 MB 到几十 MB。

### 2. 在多线程中如何保持数据一致性

- 使用锁（`mutex`、`RLock`）
- 使用原子操作（如 `queue.Queue`）
- 使用线程局部存储（`threading.local`）
- 使用不可变数据或消息传递（避免共享状态）

### 4. 集群、分布式、主从、高并发，请分别介绍一下

- **集群**：多台服务器协同工作，对外表现为一个整体。
- **分布式**：系统组件部署在不同节点上，通过网络通信协作。
- **主从**：一个节点（主）负责写，其他节点（从）负责读或备份。
- **高并发**：系统能够同时处理大量请求的能力。

### 5. CQRS读写分离

CQRS（Command Query Responsibility Segregation）将系统的**写操作**（Command）和**读操作**（Query）分离到不同的模型或数据源中，优化各自性能与扩展性。

### 6. 工作中，你们关于Redis的命名规范是什么？

常见规范：

- 使用冒号分隔层级：`业务名:模块名:key名`，如 `user:profile:123`
- 全部小写，单词间用下划线或冒号
- 避免特殊字符，设置合理过期时间

### 7. 请说一下你对DDD的理解

领域驱动设计（Domain-Driven Design）是一种软件设计方法，核心是**将业务领域作为核心**：

- 统一语言（Ubiquitous Language）
- 实体、值对象、聚合、领域服务、领域事件
- 限界上下文（Bounded Context）划分
- 分层架构（接口层、应用层、领域层、基础设施层）

### 8. 数据库索引的最左前缀是什么意思？

联合索引 `(a, b, c)` 遵循最左前缀原则：查询条件必须从索引最左边的列开始，不能跳过中间列。例如：

- `WHERE a = 1` ✅ 用到索引
- `WHERE a = 1 AND b = 2` ✅
- `WHERE b = 2` ❌ 不会使用该索引

### 9. 手写一个Web框架，你会考虑哪些方面？

- 路由映射（支持动态参数、正则）
- 请求/响应对象封装
- 中间件机制（洋葱模型）
- 模板引擎或 JSON 序列化
- 静态文件服务
- 异常处理与调试模式
- 可扩展性（蓝图/插件）

### 10. 快速排序、递归的逻辑和使用场景

- **快速排序**：分治法，选基准值，将小于基准和大于基准的分到两侧，递归排序两侧。
- **递归**：函数调用自身，需有终止条件。
- **使用场景**：树/图的遍历、分治算法（快排、归并）、解析嵌套结构（JSON、XML）。

### 11. JWT中token由哪几部分组成？验证原理是什么？

- **组成部分**：`Header`（算法和类型）、`Payload`（声明数据）、`Signature`（对前两部分签名）。
- **验证原理**：服务端收到 token 后，重新用相同密钥和算法计算签名，若与 token 中的签名一致，则 token 未被篡改；再从 Payload 中提取用户信息。

### 12. 进程和线程在内存里的关系

- **进程**：拥有独立的地址空间、代码段、数据段、堆、栈。
- **线程**：属于同一进程的线程共享进程的地址空间和堆，但每个线程拥有独立的栈和寄存器上下文。
- 关系：进程是资源分配的最小单位，线程是 CPU 调度的最小单位。

### 13. 菜单权限设计中，你们采用的是3表结构还是5表结构，为什么？

- **3表结构**：用户表、角色表、权限表（或菜单表），通过用户-角色、角色-权限关联。
- **5表结构**：用户表、角色表、权限表、用户-角色关联表、角色-权限关联表（若权限含菜单和按钮，则再加资源表）。
- **选择原因**：3表适合简单系统；5表更灵活，支持更细粒度权限（如数据权限、操作按钮）。根据业务复杂度选择。
