# Python 浅拷贝与对象引用

## 核心概念

Python 里**一切皆对象**，列表的每个槽位存的是**指针（引用）**，不是值本身。

```
list → [ ptr0, ptr1, ptr2 ]
            ↓
         [object in memory]
```

## 浅拷贝（Shallow Copy）

`list(l1)` / `l1[:]` / `copy.copy(l1)` 都是浅拷贝：

- 新建一个列表对象（`id` 不同）
- 但每个槽位的引用**原样复制**，指向同一批对象

```python
l1 = [3, [4, 5, 6], (7, 8, 9)]
l2 = list(l1)

id(l1) == id(l2)   # False：两个不同的列表对象
l1[0] is l2[0]     # True：同一个整数对象 3
l1[1] is l2[1]     # True：同一个列表对象 [4,5,6]
l1[2] is l2[2]     # True：同一个元组对象 (7,8,9)
```

## 可变 vs 不可变对象的影响

| 操作 | 影响 |
|------|------|
| `l1.append(10)` | 只改 l1 自己，l2 不受影响 |
| `l1[1].remove(4)` | 改的是共享的内层列表，**l1 和 l2 都变** |
| `l2[2] += (10, 11)` | tuple 不可变，`+=` 会创建新 tuple 并重新绑定 l2[2]，**l1 不受影响** |

关键规则：

- **修改共享的可变对象**（list、dict）→ 两边都看到变化
- **重新绑定某个槽位**（赋值、+=不可变对象）→ 只影响当前列表

## 整数驻留（Interning）

Python 对 `-5 ~ 256` 的整数做缓存，内存里只有一个实例：

```python
a = 3
b = 3
a is b  # True，同一个对象
```

超出范围的整数在不同地方可能是不同对象（CPython 实现细节，非语言规范）。

## `is` vs `==`

- `is`：比较 `id()`，即内存地址，判断是否**同一对象**
- `==`：比较值，调用 `__eq__`

同一对象必然 `==` 成立；`==` 成立不代表 `is` 成立。

## 深拷贝

```python
import copy
l2 = copy.deepcopy(l1)  # 递归复制所有嵌套对象，完全独立
```
