# docker

## 构建一个image

```bash
docker compose build app
```
其中app是镜像的名字


## 基于这个这个构建的image 构建一个container

```bash
docker compose run -d \
    --name cms-center-test-shell \
    --no-deps \
    app
```

## 进入容器

```bash
docker exec -it cms-center-test-shell bash
``

进入后,就和正常的bash是一样的效果了.

比如 

```bash
php artisan test
```

## 备注:

其实正常来说不使用 `docker compose build app` 也是可以的,使用 `docker build app` 会麻烦很多,如下:

```bash
docker build \
    -f docker/Dockerfile \
    -t cms-center-image \
    .
```

含义：

- -f docker/Dockerfile：指定 Dockerfile。
- -t cms-center-image：镜像名称必须与 Compose 中一致。
- .：仓库根目录作为构建上下文。

> 推荐使用 docker compose build app，因为它会自动跟随 Compose 中的镜像名、Dockerfile 和构建上下文；将来配置变化时不必同步修改手写命令。
