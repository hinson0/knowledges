如果你想走“纯 WSL2”，现在这个已打开的 `C:\Users\EDY\Desktop\Coding\ce-workspace` 不应该继续作为主工作区。正确实践是：**在 WSL 的 Linux filesystem 里重新放一份 `ce-workspace`，然后让 Codex Desktop 的 agent 和 terminal 都跑 WSL**。

**推荐做法**

1. 在 WSL/Ubuntu 里准备目录：

```bash
mkdir -p ~/code
cd ~/code
```

1. 把 `ce-workspace` 放到 WSL home 下，目标路径类似：

```bash
~/code/ce-workspace
```

最干净是重新 clone harness 仓库：

```bash
cd ~/code
git clone <你的 ce-workspace 仓库地址> ce-workspace
cd ce-workspace
```

如果你现在 Windows 这份里面有很多未提交的 harness 文件、`projects/<repo>` source clone、`worktrees/<feature>/<repo>`，可以先做一次搬运：

```bash
rsync -a \
  --exclude node_modules \
  --exclude .venv \
  --exclude dist \
  --exclude build \
  /mnt/c/Users/EDY/Desktop/Coding/ce-workspace/ \
  ~/code/ce-workspace/
```

1. 在 WSL 里装工具，至少这些：

```bash
sudo apt update
sudo apt install -y git curl ripgrep nodejs npm python3 python3-pip bubblewrap gh
```

如果你的项目用 `pnpm`：

```bash
corepack enable
```

1. 在 Codex Desktop 里改设置：

打开 **Settings**，把 agent 从 **Windows native** 切到 **WSL**，然后 **重启 Codex app**。同时建议把 integrated terminal 也设成 **WSL**，这样 Codex 跑命令和你看到的 terminal 是同一个世界。

1. 重新添加 WSL 里的项目：

在 Codex Desktop 里 **Add new project** / `Ctrl+O`，文件选择框输入：

```text
\\wsl$\
```

然后进入你的发行版，比如：

```text
\\wsl$\Ubuntu\home\<你的 WSL 用户名>\code\ce-workspace
```

打开这个 WSL 版本的 `ce-workspace`。

1. 新线程里确认环境：

让 Codex 或你在 terminal 跑：

```bash
pwd
echo $WSL_DISTRO_NAME
git status
node scripts/harness.mjs status <feature>
```

理想输出特征是：

```text
/home/<user>/code/ce-workspace
Ubuntu
```

不是：

```text
C:\Users\EDY\Desktop\Coding\ce-workspace
```

也不是：

```text
/mnt/c/Users/EDY/Desktop/Coding/ce-workspace
```

对你这个 harness，我会把核心规矩定成一句话：**纯 WSL2 = repo 在 `/home/...`，Codex agent 在 WSL，terminal 在 WSL，Git/Node/Python/测试全部在 WSL。** 现在这个 Windows 路径可以先保留作备份，但不要两边同时开发同一个 feature，尤其不要两边都碰 `worktrees/<feature>`，那会很快乱起来。
